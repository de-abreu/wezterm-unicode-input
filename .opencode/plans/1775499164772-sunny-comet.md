# Fix wezterm unicode-input module error

## Problem
The error occurs on line 15 of wezterm.lua:
```
require("modules.unicode-input").apply_to_config(config)
```

The module `/home/abreu/.config/wezterm/modules/unicode-input.lua` returns `config` (line 36) instead of the module, so `.apply_to_config` is called on the wrong object.

## Root Cause
In `unicode-input.lua`:
- Line 15: Calls `unicode_input.apply_to_config(config, {...})` internally - this works
- Line 36: Returns `config` - causes the error when required externally

When wezterm.lua requires the module, it gets `config` (the WezTerm config builder) instead of the module table. `config` has no `apply_to_config` method.

## Fix
Edit `/home/abreu/.config/wezterm/modules/unicode-input.lua`:
- Remove or comment out line 36 (`return config`)

Since the module already invokes `apply_to_config` internally (line 15), it doesn't need to return anything. Alternatively, return the module: `return module`.

## Verification
1. Restart wezterm or run `:reload` in the console
2. The error should be gone
